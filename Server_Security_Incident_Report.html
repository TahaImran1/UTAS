1. Incident Summary

Root Cause Summary: The server was placed in the DMZ (or port range forwarding was configured) on the router on June 15, exposing all internal ports to the public internet. Because the Windows Firewall was disabled (pre-existing condition), automated internet scanners targeted a vulnerability in Oracle WebLogic (Port 7001) on June 16 at 4:00 AM, dropping a renamed XMRig Monero Miner (v6.21.2).

The malware established persistence by creating a registry-based Windows Service named JavaSE. The service launched a renamed copy of NSSM (Non-Sucking Service Manager) from AppData\Roaming\Sun\Java\Deployment\java.exe, which spawned the actual XMRig CPU miner renamed as java.exe at AppData\LocalLow\Sun\Java\Deployment\java.exe, causing a 90%+ CPU utilization spike.

2. Diagnostic & Discovery Commands

The following commands were utilized to inspect process information and verify the security status of the server:

Identify Active Modules & Bitness Mismatch

Used to verify if the spinning process was a legitimate JVM (by checking if the Java Virtual Machine DLL is loaded) or a fake binary:

Get-Process -Id <PID> | Select-Object -ExpandProperty Modules | Where-Object {$_.ModuleName -like "*jvm.dll*"} | Select-Object FileName
What it does: Searches the loaded DLLs of the target process ID. If it returns blank, it confirms the binary is not a real Java Virtual Machine.
Check Security Auditing Status

Used to verify whether Windows was logging administrative and firewall changes to the Security log:

auditpol /get /subcategory:"MPSSVC Rule-Level Policy Change"
What it does: Checks if the subcategory policy is set to "Auditing" or "No Auditing". (It was set to No Auditing on the server).
Check Firewall Service Controls (System Log)

Used to search for Service Control events related to the Windows Defender Firewall service starting or stopping:

Get-EventLog -LogName System -Source "Service Control Manager" -ErrorAction SilentlyContinue | Where-Object {$_.Message -like "*Windows Defender Firewall*"} | Select-Object TimeGenerated, Message
What it does: Searches the System logs for service controller events targeting the firewall service.

3. Remediation & Cleanup Commands

The following commands were executed to terminate the active malware, delete the persistent service, and clean up the malicious files:

1. Terminate Malicious Processes
Stop-Process -Id 3216 -Force
Stop-Process -Id 8328 -Force
What it does: Instantly stops the parent service manager (3216) and the child miner (8328), dropping CPU usage back to normal.
2. Delete the Persistent Windows Service
Remove-Item -Path "HKLM:\System\CurrentControlSet\Services\JavaSE" -Recurse -Force
What it does: Deletes the service configuration key directly from the registry, preventing the malware from restarting on reboot (bypassing SCM access denial).
3. Remove Fake Binary Files
Remove-Item -Path "C:\Users\Administrator\AppData\Roaming\Sun\Java\Deployment\java.exe" -Force
Remove-Item -Path "C:\Users\Administrator\AppData\LocalLow\Sun\Java\Deployment\java.exe" -Force
What it does: Deletes the masquerading service manager and miner executables from the user AppData directories.

4. Security Hardening & Monitoring Setup

The following commands were run to enable defensive structures on the server and monitor future activity:

Enable Windows Defender Firewall
Set-NetFirewallProfile -All -Enabled True
What it does: Turns the Windows Defender Firewall ON for all network profiles (Domain, Private, and Public). This immediately blocked public access to WebLogic (7001) and Oracle Database (1522).
Enable Firewall Policy Security Auditing
auditpol /set /subcategory:"MPSSVC Rule-Level Policy Change" /success:enable /failure:enable
What it does: Configures Windows to write an audit log entry to the Security Event Log every time the firewall status is toggled or rules are altered.
Query Security Audit Logs for Firewall Changes
Get-WinEvent -FilterHashtable @{LogName='Security'; Id=4950} -ErrorAction SilentlyContinue | Select-Object TimeCreated, Id, Message | Format-List
What it does: Pulls and displays the log history of firewall toggles (Event ID 4950). Verified successfully when testing the firewall state changes.

5. Recommended Network Configuration

Status Update: The firewall has successfully closed WebLogic (7001) and Oracle XE (1522) from the internet. However, Port 1521 (Oracle DB) and Port 3389 (RDP) remain open because of active router forwarding rules.

To complete the server hardening, configure the router settings to ensure maximum isolation:

  • Disable DMZ entirely on the router.
  • Set specific port forwarding rules mapping only Port 4370 (TCP and UDP) to the server for ZKTeco syncing.
  • Remove/Delete any router rules forwarding 3389 (RDP), 1521 (Oracle Database), 8888, and 8085 (Tomcat) to the public internet.
  • Utilize a secure, encrypted peer-to-peer VPN (e.g., Tailscale) for RDP/Database connections rather than exposing ports directly.
